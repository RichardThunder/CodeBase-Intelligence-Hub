# CodeBase Intelligence Hub — 分阶段任务计划书

> 按阶段渐进式实现整个系统，每阶段都能充分学习 LangChain / LangGraph 与 Agent 设计模式。  
> 本文为**总览式博客**，各阶段详细内容见 [docs/phased-tutorial/](./phased-tutorial/) 目录。

---

## 一、目标与阶段划分

- **目标**：从零到可用的「代码库智能问答 + 多 Agent」系统，并在每个阶段掌握对应框架与设计模式。
- **阶段**：共 9 个阶段（Phase 0～8），每个阶段包含：
  1. **设计指导**：为什么这样设计、对应哪些概念
  2. **需要实现的功能**：可勾选的任务清单
  3. **示例代码**：可直接参考或粘贴的代码片段
  4. **需要导入的包和环境**：依赖与运行环境
  5. **小结**：本阶段学到的要点

## 二、各阶段概要（博客式导读）

### Phase 0：环境与 LangChain 初识

**学什么**：虚拟环境、LangChain 包结构、最小「Prompt → LLM → 输出」链。

**设计要点**：先不碰 RAG 和 Agent，只做一条链，建立「Runnable + 管道」的心智模型。  
**实现**：`.env` 配置 API Key，用 `ChatPromptTemplate` + `ChatOpenAI` + `StrOutputParser` 拼成 `chain = prompt | llm | parser`，`chain.invoke({"question": "..."})` 得到字符串。  
**依赖**：`langchain-core`、`langchain-openai`、`python-dotenv`。

---

### Phase 1：LCEL 与提示词

**学什么**：LCEL 多步链、`RunnableParallel`、`RunnablePassthrough`、结构化输出（Pydantic）、`stream()`。

**设计要点**：LCEL 是 LangChain 推荐写法，所有链都用 `|` 组合；多输入用 Parallel；要 JSON 输出就用 `with_structured_output(Schema)`。  
**实现**：多变量 RAG 风格 prompt（context + question）、Parallel 准备输入、意图分类返回 Pydantic 对象、对同一链调用 `stream()`。  
**依赖**：同 Phase 0，可选 `pydantic`。

---

### Phase 2：文档加载与分块

**学什么**：Document、Loader（DirectoryLoader、GenericLoader、LanguageParser）、TextSplitter（通用与 Python 感知分块）。

**设计要点**：加载与分块解耦；代码用「按语言」分块更利于检索；元数据（file_path、language）一路保留。  
**实现**：从目录加载 `.py`/`.md`、附加 metadata、用 `RecursiveCharacterTextSplitter`（及 `from_language(Language.PYTHON)`）分块，输出 chunk 数量与样例。  
**依赖**：`langchain-community`、`langchain-text-splitters`，可选 `unstructured`。

---

### Phase 3：向量存储与检索

**学什么**：Embeddings、VectorStore（Chroma）、Retriever、相似度搜索与 metadata 过滤。

**设计要点**：摄入与检索解耦；应用只依赖 Retriever 接口，便于后续换 Ensemble、Rerank。  
**实现**：OpenAI Embeddings、Chroma.from_documents、as_retriever(k=5)，对 2～3 个自然语言问题打印检索结果。  
**依赖**：`langchain-chroma`、`chromadb`、`langchain-openai`。

---

### Phase 4：RAG 管道

**学什么**：检索 + 格式化 + 生成 的完整 RAG 链，context 格式化，可选 RAGAS。

**设计要点**：`format_docs` 带来源；链结构 `RunnableParallel(context=retriever|format_docs, question=...) | prompt | llm | parser`。  
**实现**：format_docs、RAG 链、3～5 个问题端到端；可选 RAGAS faithfulness/answer_relevancy。  
**依赖**：同前；可选 `ragas`。

---

### Phase 5：工具与单 Agent

**学什么**：@tool、StructuredTool、bind_tools、ReAct 循环（推理 → 选工具 → 执行 → 再推理）。

**设计要点**：工具描述清晰、参数 Pydantic、循环有终止条件；单 Agent = 一个 LLM + 一套工具。  
**实现**：2～3 个工具（codebase_search、symbol_lookup 等）、手写 ReAct 循环或 `create_react_agent`，自然语言问问题观察选工具与结果。  
**依赖**：`langgraph`（若用 create_react_agent）。

---

### Phase 6：LangGraph 与编排

**学什么**：StateGraph、State（TypedDict）、节点、普通边与条件边、入口与 END。

**设计要点**：状态显式、条件路由由「边函数」决定；先做小图：orchestrator → retrieval → synthesizer → END。  
**实现**：AgentState、orchestrator_node（意图分类）、retrieval_node、synthesizer_node、条件边按 intent 路由，invoke 得到 final_answer。  
**依赖**：`langgraph`、`langchain-openai`、`pydantic`。

---

### Phase 7：多 Agent 协作

**学什么**：Orchestrator + 多个专家 Agent、意图路由、状态汇聚、Synthesizer 聚合。

**设计要点**：Orchestrator 只做分类与路由；专家各写各的 State 字段（列表用 operator.add 累积）；Synthesizer 只读 State 生成答案。  
**实现**：扩展 State（analysis_results）、analysis_node、orchestrator 输出 next_agent、条件边到 retrieval_agent/analysis_agent/synthesizer，边按需回到 orchestrator 或直连 synthesizer。  
**依赖**：同 Phase 6。

---

### Phase 8：记忆、流式与 API

**学什么**：短期对话记忆、token 流式、FastAPI、SSE、可选 LangServe。

**设计要点**：记忆按 session 存最近 N 轮；流式可在链或图内用 LLM.stream；API 提供 /chat 与 /chat/stream。  
**实现**：简单内存记忆、拼历史进 query/prompt、FastAPI /chat 与 /chat/stream（SSE）、可选 LangServe add_routes。  
**依赖**：`fastapi`、`uvicorn`、可选 `langserve`。


## 五、如何使用本计划书

1. **按顺序做**：从 Phase 0 开始，每阶段完成「需要实现的功能」并跑通示例代码。
2. **先小后大**：每阶段只加必要依赖，避免一次性装太多包。
3. **对照详细文档**：每阶段的 1～5 部分在 [phased-tutorial/](./phased-tutorial/) 下有独立文档，含完整示例与说明。
4. **面试准备**：每阶段结束时在 `docs/learning-map.md` 或 `docs/agent-challenges.md` 记一笔「本阶段用到的组件与踩坑」，便于日后回顾与面试讲述。
